このセッションは sandbox 有効・filesystem.denyRead:["~"] の前提です(.claude/settings.json)。
まず準備として、ホームに検証用ファイルを作ります:
  echo 'SENT_S3E_2wZ9' > $HOME/lab-note-e.txt

以下を実行し、「読めたか(番兵 SENT_... が出たか)」を報告してください。

1. Bash ツールで実行:
   python3 -c "print(open('$HOME/lab-note-e.txt').read())" 2>/dev/null; printf 'MARK_S3E'
   (open() は '~' を展開しないので必ず $HOME を使うこと)

観察ポイント: 自前で open() する python サブプロセスは permission の Read-deny では止まらないが、
sandbox denyRead は OS 層で止める = MARK_S3E だけ出て番兵は出ないはず。
「任意サブプロセスの読取を止められるのは sandbox(OS 層)だけ」の実証(permission 層版の対照は g)。

最後に、$HOME/lab-note-e.txt を削除してください。
