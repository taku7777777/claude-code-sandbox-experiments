このセッションは sandbox 有効(.claude/settings.json の sandbox.enabled=true)で動いている前提です。allowWrite は設定していません。
以下の Bash コマンドを上から順に1つずつ実行し、それぞれ「承認を求められたか / 実行できたか(ファイルができたか)」を1行で報告してください。ブロックされたら他のツールで代替せず、結果だけ報告してください。

1. echo data > inside.txt                       （cwd 直下への書込）
2. echo data > $HOME/lab-fs-write/probe.txt      （cwd 外・allowWrite 未登録）
3. echo data > $HOME/lab-glob-XYZ/probe.txt      （cwd 外・別パス）

(1 は承認なしで成功、2 と 3 は承認プロンプトも出ないまま "operation not permitted"(OS サンドボックスの EPERM)で失敗するはず＝sandbox の書込境界は cwd(+tmp)だけ、が観察対象です。cwd 外は permission では通っているのに OS 層で止まる=allow ❌。
最後に、この検証で作った inside.txt と、もし出来ていれば ~/lab-fs-write ~/lab-glob-XYZ を削除してください。)
