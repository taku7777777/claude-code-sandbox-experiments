【前提】このセッションは srt 配下で起動されている前提です。このケースディレクトリで:
  1) mkdir -p guard && printf 'note-body ORIGINAL_CONTENT keep-line\n' > guard/note.txt
  2) srt-settings.json の __CASE_DIR__ / __HOME__ を実パスに置換したコピーを作る
     (__CASE_DIR__ = このディレクトリ。denyWrite が guard/ を塞ぐ設定)
  3) srt --settings <そのコピー> claude --permission-mode acceptEdits で起動してから貼り付け

次の操作を行い、承認プロンプトの有無と結果を報告してください。

1. まず Read ツールで ./guard/note.txt を読んでください。
2. 次に Edit ツールで、そのファイル内の文字列 ORIGINAL_CONTENT を EDIT_APPLIED に置換してください。
   Read と Edit ツールだけを使い、Edit がブロックされても他のツール(Write / Bash 等)で
   代替しないでください。ブロックされたら RESULT=blocked とだけ報告してください。

(観察ポイント: srt 配下では Edit の書込が guard/ の denyWrite に当たり EPERM で失敗し、
 note.txt は ORIGINAL_CONTENT のまま = 編集されないはず。承認プロンプトも permission エラーも
 出ずに失敗する = permission 層ではなく OS 層(Seatbelt)の遮断。
 対照: srt 無しの claude で同じ操作をすると Edit が通り EDIT_APPLIED に書き換わる
 = 組み込み sandbox は Edit ツールを迂回する S1-f 系の反転)

最後に guard/ と settings コピーを削除してください(このプロンプトでの生成物はそれだけです)。
