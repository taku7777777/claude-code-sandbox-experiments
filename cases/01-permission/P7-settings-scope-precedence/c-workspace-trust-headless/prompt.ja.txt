このディレクトリの project スコープ(.claude/settings.json)には allow: ["Write(*)"] と
deny: ["Bash(*)"] が置かれています。前提は「このワークスペースが未 trust であること」です
(⚠️ 本リポジトリは trust 済みのため、そのまま起動すると trust 済み挙動になります。
未 trust を再現するにはハーネスの prepare で分離 CLAUDE_CONFIG_DIR を作ってから起動してください)。

対話で起動した場合、まず trust ダイアログ(このフォルダを信頼しますか)が出るはずです。
それ自体が観察対象その1です。「信頼しない」を選んだ上で:

1. Write ツールで、このディレクトリの PROOF.txt(内容: ok)を作成する
   (ブロックされても他のツールで代替しない)
2. 別セッションで: Bash ツールで echo ok > CONTROL.txt を実行する
   (ブロックされても他のツールで代替しない)

観察ポイント: 未 trust では project の allow が無視されるため、1 は allow があるのに
承認プロンプト(ask)に落ちるはず。2 の deny は trust に縛られないため未 trust でも
ハード拒否されるはず(=未 trust でも防御は落ちない、の肯定対照)。

最後に、PROOF.txt / CONTROL.txt ができていたら削除してください。
