このディレクトリの設定は allow: ["Bash(*)"] + deny: ["Bash(curl:*)"] です(広い allow の内側に狭い deny)。
以下の操作を上から順に1つずつ実行し、それぞれ「承認を求められたか / 拒否されたか / 実行できたか」を1行で報告してください。
Bash が拒否されても他のツールで代替しないでください。

1. Bash ツールで、次のコマンドをそのまま実行する: curl -sS https://example.com -o CURLED.txt
2. Bash ツールで、次のコマンドをそのまま実行する: echo ok > ECHOED.txt

(観察対象: 1 の curl はプロンプトなしで拒否される(狭い deny が広い allow に勝つ)。
2 の echo は deny にマッチしないため Bash(*) の allow がそのまま効いて実行される。
= 狭い deny は「マッチした形」の分だけ例外(穴)を彫れる、が観察対象です。
最後に、作成された CURLED.txt / ECHOED.txt があればすべて削除してください。)
